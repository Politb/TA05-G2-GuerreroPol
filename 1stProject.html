Fail2Ban + Discord Integration

This project demonstrates the integration of the Fail2Ban intrusion detection system with Discord, allowing real-time security notifications directly to a Discord channel. The system detects brute-force attacks, DoS attempts, and unauthorized access attempts on services such as SSH and Apache2.

Objectives

  • Configure Fail2Ban as an IDS to detect brute-force and DoS attacks.
  • Integrate Fail2Ban with Discord using Webhooks.
  • Simulate attacks using Apache Benchmark (ab) and SSH brute-force attempts.
  • Automatically notify a Discord channel when Fail2Ban bans or unbans an IP.

Software Used

  • Fail2Ban
  • Apache2
  • OpenSSH Server
  • Apache2-utils (ab)
  • Discord Webhooks
  • Ubuntu Linux (Server + Client VMs)

Key Features

  • Automatic IP banning after failed SSH login attempts.
  • Detection of DoS attacks using custom filters.
  • Custom Fail2Ban action script to send Discord notifications.
  • Real-time alerts for: service start, stop, ban, and unban events.
